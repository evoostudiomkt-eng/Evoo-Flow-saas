# Developer & Agent Instructions for Client-Agencies Project

This file contains rules and constraints that **must** be strictly followed by any AI agent or developer continuing work on this codebase.

## 📌 Data Retention and Protection Rule

* **DO NOT DELETE OR CLEAR REGISTERS OF AGENCIES:** 
  The user is starting true/real-world validation and tests as an Agency Owner. 
  * Any agency registered or created in the system **MUST BE DEEMED DEFINITIVE AND PERMANENT**.
  * Never run scripts, batch updates, or write code that wipes, modifies, or deletes existing collections of agencies, clients, or configuration details in the Firebase Firestore Database unless explicitly requested by the user.
  * Respect all manually created administrative accounts and active client portfolios.
